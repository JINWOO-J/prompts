# Analogical Reasoning — 유추 추론

## 개념

다른 도메인의 유사한 문제/해결책을 현재 문제에 적용하는 기법.
'A:B = C:?' 형태의 구조적 유사성을 활용.
Gentner(1983)의 구조 매핑 이론(Structure Mapping Theory)이 이론적 기반.
혁신적 해결책은 종종 다른 분야의 유추에서 나온다 (예: 벨크로 = 도꼬마리 열매).

## 핵심 원리

- 구조적 유사성: 표면적 유사성이 아닌 관계 구조의 유사성이 핵심
- 지식 전이: 잘 아는 도메인의 지식을 새로운 도메인에 적용
- 창의적 도약: 논리적 추론으로는 도달하기 어려운 해결책 발견
- 커뮤니케이션: 복잡한 개념을 친숙한 비유로 설명하면 이해도 향상

## 프롬프트 템플릿

```
**현재 문제:** [해결할 문제]

**유추 탐색:**

**유추 1 — [도메인 A]에서의 유사 문제:**
- 원본 문제: [도메인 A의 문제]
- 원본 해결책: [도메인 A의 해결 방법]
- 구조적 유사성: [어떤 구조가 비슷한가]
- 우리 문제에 적용: [구체적 적용 방안]

**유추 2 — [도메인 B]에서의 유사 문제:**
- 원본 문제: [도메인 B의 문제]
- 원본 해결책: [도메인 B의 해결 방법]
- 구조적 유사성: [어떤 구조가 비슷한가]
- 우리 문제에 적용: [구체적 적용 방안]

**구조적 매핑 테이블:**
| 원본 도메인 요소 | 현재 문제 요소 | 매핑 근거 |
|----------------|-------------|----------|
| [요소 A] | [대응 요소] | [...] |
| [요소 B] | [대응 요소] | [...] |
| [해결 메커니즘] | [적용 방안] | [...] |

**유추의 한계:**
- 매핑이 깨지는 지점: [어디서 유추가 더 이상 성립하지 않는가]
- 원본과 다른 제약: [현재 문제만의 고유한 제약]
- 보완 필요 사항: [유추만으로 부족한 부분]
```

## 실전 예시

**시나리오: "마이크로서비스 간 통신 장애 처리"**

유추 1 — 전력망(Power Grid):
- 원본: 발전소 하나가 다운되면 다른 발전소가 부하를 분담
- 적용: Circuit Breaker 패턴 — 장애 서비스를 차단하고 fallback으로 전환

유추 2 — 면역 체계:
- 원본: 바이러스 감지 → 격리 → 항체 생성 → 기억
- 적용: 이상 탐지 → 서비스 격리 → 자동 복구 → 패턴 학습(AIOps)

구조 매핑: 발전소=서비스, 전력망=서비스 메시, 차단기=Circuit Breaker

## 변형 및 조합

- **Analogy + First Principles**: 유추 후 제1원리로 검증
- **Analogy + Feynman**: 유추를 활용한 설명으로 이해도 확인
- **Analogy + Lateral Thinking**: 의도적으로 먼 도메인에서 유추 탐색

## 주의사항

- 표면적 유사성에 속을 수 있음 (구조적 유사성을 확인해야 함)
- 유추가 깨지는 지점을 인식하지 못하면 잘못된 결론에 도달
- 너무 먼 도메인의 유추는 실용성이 떨어질 수 있음
- 유추는 가설 생성 도구이지 증명 도구가 아님 — 별도 검증 필요

## 적합한 상황

혁신적 문제 해결, 아키텍처 설계, 설명/교육, 크로스 도메인 학습, 브레인스토밍
