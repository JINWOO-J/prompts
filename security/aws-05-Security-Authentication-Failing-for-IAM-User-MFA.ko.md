# IAM 사용자 MFA 인증 실패 — MFA Authentication Failing for IAM User

## 의미

IAM 사용자의 MFA 인증이 실패합니다(인증 오류 또는 IAMMFAFailure 알람 발생). 원인으로는 MFA 디바이스 미구성, MFA 토큰 무효 또는 만료, MFA 디바이스 분실 또는 등록 해제, IAM 정책에서 MFA를 요구하지만 디바이스 미등록, MFA 토큰 검증 서비스 오류, 디바이스 시간 동기화 문제로 인한 토큰 불일치 등이 있습니다. IAM 사용자가 인증할 수 없고, AWS 서비스 접근이 거부되며, MFA 필수 작업이 실패합니다. 이는 인증 및 인가 계층에 영향을 미치며 사용자 접근을 차단합니다. 일반적으로 MFA 디바이스 구성 문제, 토큰 검증 문제, 정책 설정 오류가 원인이며, 하드웨어 MFA 디바이스를 사용하는 경우 디바이스 문제가 인증에 영향을 줄 수 있고 애플리케이션에서 사용자 접근 실패가 발생할 수 있습니다.

## 영향

IAM 사용자 인증 불가, AWS 서비스 접근 거부, MFA 필수 작업 실패, 사용자 로그인 시도 차단, 인증 오류 발생, IAM 사용자 세션 수립 불가, MFA 보호 리소스 접근 불가, 사용자 접근 완전 차단. IAMMFAFailure 알람 발생 가능. 하드웨어 MFA 디바이스를 사용하는 경우 배터리 또는 시간 동기화 문제로 실패 가능. 차단된 사용자 접근으로 인해 애플리케이션 오류나 성능 저하 발생 가능. 사용자 대면 서비스가 접근 불가능해질 수 있습니다.

## 플레이북

1. IAM 사용자 `<user-name>`의 존재를 확인하고 리전 `<region>`의 IAM AWS 서비스 상태가 정상인지 확인합니다.
2. IAM 사용자 `<user-name>`을 조회하여 MFA 디바이스 구성, MFA 디바이스 상태, MFA 활성화 상태를 검사하고 MFA 디바이스가 활성 상태인지 확인합니다.
3. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹을 조회하여 사용자 `<user-name>`과 관련된 MFA 인증 실패 이벤트를 필터링하고 인증 시도 패턴을 포함합니다.
4. 사용자 `<user-name>`의 IAM 사용자 로그인 시도를 조회하여 MFA 인증 실패 패턴이나 오류 메시지를 확인하고 실패 원인을 분석합니다.
5. 사용자 `<user-name>`에 연결된 IAM 정책을 나열하고 정책 문에서 MFA 요구 조건을 확인하며 정책 조건 구문을 검증합니다.
6. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹을 조회하여 사용자 `<user-name>`의 MFA 디바이스 등록 또는 삭제 이벤트를 필터링하고 디바이스 생명주기 이벤트를 확인합니다.
7. IAM 사용자 `<user-name>`의 MFA 디바이스 시리얼 번호 또는 가상 MFA 디바이스 ARN을 조회하여 디바이스 상태를 확인하고 디바이스가 동기화되었는지 확인합니다.
8. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹을 조회하여 지난 24시간 동안 사용자 `<user-name>`과 관련된 IAM 정책 수정 이벤트를 필터링하고 MFA 요구사항 변경을 확인합니다.
9. MFA 실패율을 포함한 IAM 인증 시도에 대한 CloudWatch 메트릭을 지난 1시간 동안 조회하여 인증 실패 패턴을 식별합니다.

## 진단

1. CloudTrail 이벤트(플레이북 3단계 및 4단계)를 분석하여 MFA 인증 실패가 처음 발생한 시점을 식별합니다. 이 타임스탬프가 근본 원인 분석을 위한 상관관계 기준선을 수립합니다.

2. CloudTrail에서 실패 타임스탬프 전후로 MFA 디바이스 삭제 이벤트(플레이북 6단계)가 확인되면, IAM 정책이 여전히 MFA를 요구하는 상태에서 디바이스가 제거되어 인증 실패가 발생한 것입니다.

3. IAM 정책(플레이북 5단계)에서 MFA 조건 요구사항이 확인되지만 사용자 MFA 구성(플레이북 2단계)에서 활성 디바이스가 없는 경우, 정책은 MFA를 요구하지만 디바이스가 구성되지 않았거나 활성화되지 않은 것입니다.

4. MFA 디바이스가 구성되어 있고 활성 상태(플레이북 2단계)이지만 실패가 지속되면, 디바이스 시리얼 번호와 동기화 상태(플레이북 7단계)를 확인합니다. 디바이스와 AWS 간의 시간 동기화 문제가 토큰 검증 실패를 유발합니다.

5. 실패가 여러 사용자에게 영향을 미치는 경우(플레이북 3단계의 CloudTrail 분석), 계정 전체 패턴을 확인합니다. 특정 사용자만 영향받으면 사용자별 문제(디바이스 또는 정책)이고, 광범위하면 MFA 서비스 가용성이 원인일 수 있습니다.

6. CloudTrail의 IAM 정책 수정 이벤트(플레이북 8단계)에서 최근 MFA 요구사항 추가가 확인되면, 정책 적용 전에 사용자가 MFA 디바이스를 올바르게 구성했는지 확인합니다.

7. 실패 빈도가 지속적 실패가 아닌 간헐적 패턴을 보이면(플레이북 9단계), 토큰 타이밍 문제 또는 일시적 서비스 가용성 문제가 원인일 가능성이 높습니다.

상관관계가 발견되지 않는 경우: 분석을 48시간으로 확장하고, 하드웨어 MFA 디바이스 배터리 상태를 검토하고, 가상 MFA 애플리케이션 시간 동기화를 확인하고, MFA 서비스 상태를 확인하고, 엣지 케이스에 대한 IAM 정책 조건 평가 로직을 조사합니다.
