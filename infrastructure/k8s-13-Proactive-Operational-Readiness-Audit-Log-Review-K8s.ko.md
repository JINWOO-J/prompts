# Audit Log Review — 감사 로그 검토

## 의미

감사 로그 검토는 Kubernetes 감사 로그를 검토할 수 없거나 감사 로그 문제가 감지되었음을 나타냅니다(AuditLogReviewFailed 또는 AuditLogUnavailable 같은 알림 발생).
 감사 로그 검토 도구가 실패하거나, 감사 로그가 사용 불가하거나, 감사 로그 분석에서 문제가 감지되거나, 감사 로그 구성이 누락되거나, 감사 로그 모니터링에서 문제가 감지되는 것이 원인입니다. 감사 로그 검토에서 실패가 나타나고, 감사 로그가 사용 불가하며, 감사 로그 분석에서 문제가 표시되고, 감사 로그 검토가 실패합니다. 이는 컴플라이언스 계층과 감사 추적 관리에 영향을 미치며, 일반적으로 감사 로그 구성 실패, 감사 로그 검토 도구 실패, 감사 로그 수집 문제, 또는 감사 로그 모니터링 격차로 인해 발생합니다. 감사 로그 검토가 컨테이너 워크로드에 영향을 미치면 컨테이너 감사 로그가 사용 불가하고 애플리케이션에서 감사 추적 격차가 발생할 수 있습니다.

## 영향

AuditLogReviewFailed 알림 발생, AuditLogUnavailable 알림 발생, 감사 로그 검토 불가, 감사 로그 문제 감지, 감사 추적 관리 저하 가능, 컴플라이언스 감사 제한 가능. 감사 로그 검토에서 실패가 나타나며, 감사 로그 검토가 컨테이너 워크로드에 영향을 미치면 컨테이너 감사 로그가 사용 불가하고, Pod 감사 추적이 불완전하며, 컨테이너 애플리케이션에서 감사 추적 격차가 발생할 수 있습니다. 애플리케이션에서 감사 로그 검토 격차 또는 컴플라이언스 감사 제한이 발생할 수 있습니다.

## 플레이북

1. kube-system 네임스페이스에서 component=kube-apiserver 레이블의 Pod를 상세 조회하여 감사 로그 구성과 정책 설정을 확인합니다.

2. kube-system 네임스페이스의 최근 이벤트를 타임스탬프 순으로 조회하여 감사 로깅 컴포넌트의 최근 문제를 확인합니다.

3. kube-system 네임스페이스의 ConfigMap audit-policy를 YAML 출력으로 조회하여 감사 정책 상태와 로그 전달 구성을 확인합니다.

4. 네임스페이스 <namespace>에서 app=audit-log-reviewer 레이블의 감사 로그 검토 Pod 로그를 조회하고 감사 로그 분석 패턴 또는 검토 실패를 필터링합니다.

5. 지난 7일 동안의 감사 로그 서비스에 대한 Prometheus 메트릭(log_file_delivery, log_file_validation 포함)을 조회하여 감사 로그 전달 문제를 확인합니다.

6. 감사 로그 파일 무결성 검증 결과를 조회하고 로그 파일 무결성 및 검증 상태를 확인합니다.

7. 감사 로그 분석 타임스탬프와 감사 로그 이벤트 타임스탬프를 1시간 이내로 비교하고, 감사 로그 데이터를 보조 증거로 사용하여 감사 로그 분석이 이벤트를 올바르게 처리하는지 확인합니다.

8. 감사 로그 스냅샷을 조회하고 감사 로그 검토를 위한 스냅샷 가용성을 확인합니다.

## 진단

1. 1단계와 3단계의 API 서버 감사 구성을 검토합니다. 감사 정책이 누락되거나 잘못 구성되면 감사 로그가 올바르게 생성되지 않습니다. 감사 정책 규칙이 필요한 작업을 포함하는지 확인합니다.

2. 5단계의 감사 로그 전달 메트릭을 분석합니다. log_file_delivery 또는 log_file_validation에서 실패가 나타나면 로그 전송 인프라가 문제입니다. 전달이 정상이면 검토 도구 분석으로 진행합니다.

3. 6단계의 로그 무결성 검증에서 실패가 나타나면 감사 로그가 손상되었거나 변조되었을 수 있습니다. 이는 즉각적인 조사가 필요한 중요한 보안 문제입니다.

4. 4단계의 감사 로그 검토 Pod 로그를 검토합니다. 로그에서 분석 패턴이나 검토 실패가 나타나면 실패가 로그 형식 문제, 접근 권한, 또는 도구 문제로 인한 것인지 확인합니다.

5. 8단계의 스냅샷 가용성에서 격차가 나타나면 과거 감사 커버리지가 불완전합니다. 감사 로그 보존 및 백업 정책이 올바르게 구성되어 있는지 확인합니다.

분석이 결론에 도달하지 못하는 경우: 2단계의 이벤트에서 감사 로깅 컴포넌트 문제를 확인합니다. 7단계의 감사 로그 분석 타임스탬프를 비교하여 이벤트 처리가 정상 작동하는지 확인합니다. 검토 실패가 모든 감사 소스에 영향을 미치는지(인프라 문제 시사) 또는 특정 소스에만 영향을 미치는지(소스별 구성 문제 시사) 판단합니다.
