# Self-Consistency — 다중 경로 다수결

## 개념

동일한 문제에 대해 여러 독립적인 추론 경로를 생성한 뒤,
가장 빈번하게 도출되는 답을 최종 답으로 선택하는 기법.
Wang et al. (2022) 제안. 단일 CoT보다 안정적이고 신뢰도 높은 결과를 제공한다.
앙상블 학습의 아이디어를 프롬프트 레벨에서 구현한 것.

## 핵심 원리

- 다양성을 통한 견고성: 서로 다른 추론 경로가 같은 답에 도달하면 신뢰도가 높음
- 편향 감소: 단일 경로의 우연한 오류를 다수결로 상쇄
- Temperature 조절: 높은 temperature(0.7-1.0)로 다양한 경로 생성
- GSM8K에서 CoT 대비 +17.9% 정확도 향상

## 프롬프트 템플릿

```
다음 문제를 3가지 서로 다른 관점에서 독립적으로 풀어주세요.
각 경로는 이전 경로를 참고하지 말고 독립적으로 추론하세요.

**문제:** [문제 설명]

**경로 1 (관점: [A]):**
[추론 과정] → 결론: [X]

**경로 2 (관점: [B]):**
[추론 과정] → 결론: [Y]

**경로 3 (관점: [C]):**
[추론 과정] → 결론: [Z]

**다수결 판정:**
- 결론 분포: X=N회, Y=N회, Z=N회
- 최종 답: [가장 빈번한 결론]
- 일치도: [N/3]
- 불일치 원인 분석: [경로 간 차이가 발생한 이유]
```

## 실전 예시

**시나리오: "이 API 응답 지연의 원인은?"**

경로 1 (네트워크 관점): DNS 조회 지연 → 결론: DNS 캐시 미스
경로 2 (DB 관점): 슬로우 쿼리 → 결론: 인덱스 누락
경로 3 (앱 관점): N+1 쿼리 패턴 → 결론: 인덱스 누락 + N+1

다수결: "인덱스 누락"이 2/3 경로에서 언급 → 1차 원인으로 확정
추가 조사: N+1 패턴도 병행 확인 필요

## 변형 및 조합

- **Self-Consistency + CoT**: 각 경로를 CoT로 상세 추론 (기본 조합)
- **Self-Consistency + Role**: 각 경로를 다른 전문가 역할로 추론
- **Self-Consistency + Temperature 변형**: 경로마다 다른 temperature 사용

## 주의사항

- 경로 수가 많을수록 비용이 선형 증가 (3-5개가 실용적)
- 모든 경로가 같은 방향으로 틀릴 수 있음 (systematic bias)
- 정답이 없는 개방형 질문에는 다수결이 무의미할 수 있음
- 각 경로가 진정으로 "독립적"이려면 프롬프트 설계에 주의

## 적합한 상황

정답이 명확한 문제, 수치 계산, 분류 판단, 높은 신뢰도가 필요한 진단, 코드 버그 원인 특정
