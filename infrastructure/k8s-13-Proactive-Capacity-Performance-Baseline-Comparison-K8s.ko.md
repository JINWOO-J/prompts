# Baseline Comparison (베이스라인 비교)

## 의미

베이스라인 비교 알림은 성능 베이스라인 비교를 수행할 수 없거나 베이스라인 비교 실패가 감지된 상황(BaselineComparisonFailed 또는 BaselineUnavailable 같은 알림 발생)을 나타냅니다. 성능 베이스라인 미가용, 베이스라인 비교 도구 실패, 베이스라인 비교 결과 실패, 베이스라인 설정 누락, 또는 베이스라인 비교 모니터링 문제 등이 원인입니다.
 베이스라인 비교가 실패를 보이고, 성능 베이스라인이 미가용하며, 베이스라인 비교 도구가 실패하고, 베이스라인 비교 모니터링이 문제를 나타냅니다. 이는 성능 모니터링 계층과 베이스라인 관리에 영향을 미치며, 주로 베이스라인 데이터 미가용, 베이스라인 비교 도구 실패, 베이스라인 설정 문제, 또는 베이스라인 비교 모니터링 실패가 원인입니다. 베이스라인 비교가 컨테이너 워크로드에 영향을 미치는 경우, 컨테이너 성능 베이스라인이 미가용하고 애플리케이션이 베이스라인 비교 실패를 겪을 수 있습니다.

## 영향

BaselineComparisonFailed 알림 발생, BaselineUnavailable 알림 발생, 베이스라인 비교 수행 불가, 베이스라인 비교 실패 감지, 성능 분석 제한 가능, 베이스라인 관리 위험. 베이스라인 비교가 실패를 보입니다. 베이스라인 비교가 컨테이너 워크로드에 영향을 미치는 경우, 컨테이너 성능 베이스라인이 미가용하고, Pod 베이스라인 비교가 실패하며, 컨테이너 애플리케이션이 베이스라인 비교 실패를 겪을 수 있습니다. 애플리케이션이 베이스라인 비교 공백이나 성능 분석 제한을 경험할 수 있습니다.

## 플레이북

1. namespace <namespace>에서 Deployment <deployment-name>을 describe하여 현재 리소스 설정과 성능 설정을 파악합니다.

2. namespace <namespace>에서 최근 이벤트를 타임스탬프 순으로 조회하여 베이스라인 메트릭에 영향을 미치는 최근 변경이나 문제를 확인합니다.

3. namespace <namespace>에서 Pod 리소스 사용량을 조회하여 현재 성능 메트릭을 예상 베이스라인과 비교합니다.

4. 서비스 <service-name>의 Prometheus 메트릭(베이스라인 메트릭, 현재 성능 메트릭)을 최근 30일 동안 조회하여 베이스라인과 비교합니다.

5. namespace <namespace>에서 label app=baseline-comparison인 베이스라인 비교 Pod의 로그를 조회하고 베이스라인 비교 실패를 필터링합니다.

6. 베이스라인 비교 도구 실행 결과를 조회하고 도구 가용성과 비교 완료 상태를 확인하여 베이스라인 비교 도구 상태를 점검합니다.

7. 서비스 <service-name>의 성능 베이스라인 이력을 조회하고 베이스라인 데이터 완전성과 베이스라인 계산 정확도를 확인하여 베이스라인 데이터 품질을 점검합니다.

8. 서비스 <service-name>의 Prometheus 알림을 조회하고 알림 베이스라인 설정을 확인하여 베이스라인이 없는 알림을 식별합니다.

## 진단

1. 3단계의 현재 성능 메트릭과 4단계의 베이스라인 메트릭을 비교합니다. 현재 메트릭이 베이스라인 임계값을 크게 초과하면 실제 성능 편차가 존재합니다. 메트릭이 베이스라인 범위 내이면 비교 인프라가 문제일 수 있습니다.

2. 6단계의 베이스라인 비교 도구 상태를 분석합니다. 도구가 실패나 미완료 상태를 보이면 비교 인프라가 문제입니다. 도구가 정상이면 베이스라인 데이터 품질 분석으로 진행합니다.

3. 5단계의 베이스라인 비교 로그에서 실패가 있으면 실패가 베이스라인 데이터 누락, 도구 설정 오류, 메트릭 수집 문제 중 어느 것 때문인지 확인합니다.

4. 7단계의 베이스라인 이력을 검토합니다. 베이스라인 데이터가 불완전하거나 계산 정확도가 낮으면 의미 있는 비교가 가능하기 전에 베이스라인 수립을 개선해야 합니다.

5. 8단계의 알림 베이스라인 커버리지에서 베이스라인 설정이 없는 서비스가 있으면 해당 서비스는 베이스라인과 비교할 수 없습니다. 중요 서비스의 베이스라인 수립을 우선합니다.

분석이 결론에 이르지 못하면: 2단계의 이벤트에서 베이스라인 메트릭에 영향을 미치는 변경을 확인합니다. 비교 실패가 모든 서비스에 영향을 미치는지(인프라 문제 시사) 또는 특정 서비스에만 영향을 미치는지(서비스별 베이스라인 공백 시사) 판단합니다. 베이스라인 데이터 수집이 의미 있는 비교를 위한 충분한 기간을 포함하는지 확인합니다.
