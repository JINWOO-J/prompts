# Feynman Technique — 파인만 학습법

## 개념

리처드 파인만의 학습법. 복잡한 개념을 초등학생도 이해할 수 있게 설명하면
진정한 이해에 도달한다는 원리. "설명할 수 없으면 이해하지 못한 것이다."
4단계: 개념 선택 → 가르치듯 설명 → 막히는 부분 식별 → 단순화.

## 핵심 원리

- 지식의 환상(Illusion of Knowledge) 파괴: 안다고 생각했지만 설명 못하는 부분 발견
- 능동적 학습: 수동적 읽기보다 설명하기가 기억 정착률 90% (Learning Pyramid)
- 간극 식별: 설명하다 막히는 부분이 정확히 이해가 부족한 부분
- 단순화의 힘: 복잡한 것을 단순하게 만드는 과정에서 핵심이 드러남

## 프롬프트 템플릿

```
**주제:** [설명할 개념]

**1단계 — 초등학생에게 설명하기:**
"[주제]란 쉽게 말하면..."
[전문 용어 없이, 일상 비유와 예시로 설명]

**2단계 — 막히는 부분 식별:**
설명하다 막힌 부분:
- ❓ [이해가 부족한 부분 1]: 왜 막혔는지 → [...]
- ❓ [이해가 부족한 부분 2]: 왜 막혔는지 → [...]

**3단계 — 다시 학습 후 재설명:**
[막힌 부분을 보완하여 처음부터 다시 설명]

**4단계 — 단순화 및 비유:**
- 핵심을 한 문장으로: [...]
- 가장 좋은 비유: "[주제]는 마치 [비유]와 같다. 왜냐하면..."
- 5살에게 설명한다면: [...]

**이해도 자가 점검:**
- 누군가 "왜?"라고 물으면 답할 수 있는가? [예/아니오]
- 예외 상황을 설명할 수 있는가? [예/아니오]
- 관련 개념과의 차이를 설명할 수 있는가? [예/아니오]
```

## 실전 예시

**시나리오: "쿠버네티스의 Pod란?"**

1단계: "Pod는 택배 상자 같은 거야. 앱이라는 물건을 담아서 배달하는 상자.
한 상자에 물건 하나만 넣을 수도 있고, 관련된 물건 여러 개를 같이 넣을 수도 있어."

2단계: 막힌 부분 — "왜 컨테이너를 직접 실행하지 않고 Pod로 감싸는가?"
→ 네트워크 네임스페이스 공유, 사이드카 패턴 등 추가 학습 필요

3단계: "Pod는 같은 집에 사는 룸메이트들이야. 같은 주소(IP)를 쓰고,
같은 냉장고(볼륨)를 공유하고, 항상 같이 이사(스케줄링)해."

4단계: "Pod = 함께 배포되고 함께 스케줄링되는 컨테이너 그룹의 최소 단위" 

## 변형 및 조합

- **Feynman + Socratic**: 설명 후 소크라틱 질문으로 이해 검증
- **Feynman + Analogy**: 비유를 여러 도메인에서 찾기
- **Feynman + Rubber Duck**: 러버덕에게 설명하며 디버깅

## 주의사항

- 이미 잘 아는 개념에는 불필요한 오버헤드
- 단순화 과정에서 중요한 뉘앙스가 손실될 수 있음
- 비유가 잘못되면 오해를 유발할 수 있음 (비유의 한계 명시 필요)
- 수학/물리 등 형식적 정의가 중요한 분야에서는 단순화에 한계

## 적합한 상황

개념 학습, 기술 문서 작성, 팀 교육, 온보딩 자료, 복잡한 시스템 설명, 면접 준비
