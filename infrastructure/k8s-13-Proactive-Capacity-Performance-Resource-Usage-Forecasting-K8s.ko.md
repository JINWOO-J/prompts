# Resource Usage Forecasting (리소스 사용량 예측)

## 의미

리소스 사용량 예측 알림은 미래 리소스 요구를 정확히 예측할 수 없거나 리소스 용량 계획이 불충분한 상황(ResourceForecastUnavailable 또는 CapacityForecastFailed 같은 알림 발생)을 나타냅니다. 리소스 사용 추세 분석 불가, 용량 예측 모델 실패, 리소스 증가 패턴 미식별, 용량 계획 데이터 미가용, 또는 리소스 사용 예측 부정확 등이 원인입니다.
 리소스 사용 추세가 예측 불가능한 패턴을 보이고, 용량 예측 메트릭이 미가용하며, 리소스 증가 예측을 계산할 수 없고, 용량 계획 분석이 실패합니다. 이는 용량 계획 계층과 리소스 프로비저닝에 영향을 미치며, 주로 불충분한 이력 데이터, 예측 모델 실패, 리소스 사용 패턴 변화, 또는 용량 계획 도구 미가용이 원인입니다. 예측이 컨테이너 워크로드에 영향을 미치는 경우, 컨테이너 리소스 요구가 과소평가되고 애플리케이션이 용량 제약을 겪을 수 있습니다.

## 영향

ResourceForecastUnavailable 알림 발생, CapacityForecastFailed 알림 발생, 미래 리소스 요구 예측 불가, 용량 계획 불충분, 리소스 프로비저닝 부적절 가능, 용량 제약이 예기치 않게 발생 가능. 리소스 사용량 예측이 미가용합니다. 예측이 컨테이너 워크로드에 영향을 미치는 경우, 컨테이너 리소스 요구가 과소평가되고, Pod 스케일링이 불충분하며, 컨테이너 애플리케이션이 용량 제약을 겪을 수 있습니다. 애플리케이션이 용량 부족이나 예기치 않은 리소스 고갈을 경험할 수 있습니다.

## 플레이북

1. namespace <namespace>에서 Pod를 wide 출력으로 조회하여 현재 Pod 분포와 namespace 전체의 리소스 소비를 확인합니다.
2. namespace <namespace>에서 최근 이벤트를 타임스탬프 순으로 조회하여 리소스 제약, 스케일링, 용량 문제 관련 이벤트를 확인합니다.
3. namespace <namespace>에서 HorizontalPodAutoscaler를 describe하여 용량 추세 분석을 위한 HPA 설정과 현재 스케일링 상태를 점검합니다.
4. Pod CPU 사용률과 메모리 사용률의 Prometheus 메트릭을 최근 90일 동안 조회하여 사용 추세를 분석하고 증가 패턴을 식별합니다.
5. 노드 CPU 사용률과 메모리 사용률의 Prometheus 메트릭을 최근 90일 동안 조회하여 노드 사용 추세를 분석하고 용량 증가 패턴을 식별합니다.
6. namespace <namespace>의 리소스 쿼터 사용 데이터를 최근 90일 동안 조회하고 사용 추세를 분석하여 증가 패턴을 식별합니다.
7. 용량 모니터링 Pod의 로그를 조회하고 최근 90일 내 사용량 증가나 용량 제약을 나타내는 패턴을 필터링합니다.
8. 리소스 사용 추세 데이터와 용량 예측 모델 출력을 비교하여 예측이 이력 추세와 일치하는지 확인하고, Prometheus 메트릭을 보조 증거로 활용합니다.

## 진단

1. 4~5단계의 90일 CPU 및 메모리 사용률 추세를 검토합니다. 추세가 일관된 증가 패턴을 보이면 예측이 미래 요구를 예측할 수 있어야 합니다. 패턴이 매우 가변적이면 예측보다 높은 여유를 가진 반응형 스케일링이 더 적절할 수 있습니다.

2. 3단계의 HPA 설정을 분석합니다. 오토스케일러가 빈번하게 제한에 도달하면 용량 예측이 수요를 과소평가하고 있을 수 있습니다. 오토스케일러가 거의 트리거되지 않으면 현재 용량이 충분합니다.

3. 6단계의 리소스 쿼터 사용량이 제한에 접근하면 예측이 이를 식별하고 용량 확장이 계획되었는지 확인합니다. 예측이 접근하는 제한을 예측하지 못했으면 모델 보정이 필요합니다.

4. 7단계의 용량 모니터링 로그를 검토합니다. 로그에 사용량 증가나 제약 패턴이 있으면 가장 빠르게 증가하는 리소스를 식별하고 해당 리소스의 용량 계획을 우선합니다.

5. 8단계의 예측 모델 출력이 실제 추세와 크게 다르면 현재 사용 패턴에 기반하여 모델 파라미터를 조정해야 합니다.

분석이 결론에 이르지 못하면: 2단계의 이벤트에서 예측 공백을 나타내는 리소스 제약이나 스케일링 문제를 확인합니다. 예측 부정확이 특정 시간대에 집중되어 있는지(패턴 인식 문제 시사) 또는 무작위인지(근본적 모델 문제 시사) 판단합니다. 의미 있는 추세 분석을 위한 충분한 이력 데이터가 존재하는지 확인합니다.
