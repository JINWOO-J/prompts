# Network Security Audit — 네트워크 보안 감사

## 의미

네트워크 보안 감사는 네트워크 보안 구성이 잘못되었거나, 과도하게 허용적이거나, 보안 모범 사례를 위반하고 있음을 나타냅니다(NetworkSecurityAuditFailed 또는 OverlyPermissiveNetworkConfig 같은 알림 발생). 네트워크 정책에서 위반이 나타나거나, 네트워크 보안 구성이 잘못되었거나, 네트워크 보안 정책이 적용되지 않거나, 네트워크 트래픽 패턴에서 의심스러운 활동이 감지되거나, 네트워크 보안 감사 결과에서 정책 위반이 감지된 것이 원인입니다. 네트워크 보안 구성에서 위반이 나타나고, 네트워크 트래픽 패턴에서 의심스러운 활동이 감지되며, 네트워크 정책이 잘못 구성되고, 네트워크 보안 감사 결과에서 정책 위반이 감지됩니다. 이는 보안 계층과 네트워크 접근 제어에 영향을 미치며, 일반적으로 잘못 구성된 네트워크 보안 설정, 네트워크 보안 모니터링 부재, 또는 보안 정책 위반으로 인해 발생합니다. 네트워크 보안이 컨테이너 워크로드를 보호하는 경우 컨테이너 네트워크 정책이 잘못 구성되고 애플리케이션에서 보안 위험이 발생할 수 있습니다.

## 영향

NetworkSecurityAuditFailed 알림 발생, OverlyPermissiveNetworkConfig 알림 발생, 네트워크 접근이 과도하게 허용적, 보안 정책 위반, 의심스러운 네트워크 트래픽 감지, 네트워크 보안 모니터링 실패. 네트워크 보안 구성에서 위반이 나타나며, 네트워크 보안이 컨테이너 워크로드를 보호하는 경우 컨테이너 네트워크 정책이 잘못 구성되고, Pod 네트워크 접근이 과도하게 허용적이며, 컨테이너 애플리케이션에서 보안 위험이 발생할 수 있습니다. 애플리케이션에서 보안 취약점 또는 무단 네트워크 접근 위험이 발생할 수 있습니다.

## 플레이북

1. 네임스페이스 <namespace>의 NetworkPolicy <policy-name>을 상세 조회하여 네트워크 정책 규칙, 인그레스 규칙, 이그레스 규칙을 검사하고 네트워크 보안 구성을 확인합니다.
2. 네임스페이스 <namespace>의 최근 이벤트를 타임스탬프 순으로 조회하여 최근 네트워크 정책 이벤트와 보안 관련 문제를 확인합니다.
3. 네임스페이스 <namespace>의 NetworkPolicy를 YAML 형식으로 조회하여 네트워크 정책 규칙 구성을 분석하고 과도하게 허용적이거나 잘못 구성된 규칙을 확인합니다.
4. 네임스페이스 <namespace>의 Service를 상세 조회하여 서비스 네트워크 구성을 확인하고 안전하지 않은 네트워크 설정이나 잘못 구성된 서비스를 파악합니다.
5. 네트워크 정책 컨트롤러 Pod의 로그를 조회하고 최근 7일 이내의 무단 접근 시도나 포트 스캔 등 의심스러운 트래픽 패턴을 필터링합니다.
6. 네트워크 보안 컴플라이언스 점검에 대한 보안 감사 결과를 조회하고 네트워크 보안 구성 오류와 관련된 심각도 'HIGH' 또는 'CRITICAL' 결과를 필터링합니다.
7. 최근 7일간의 네트워크 트래픽에 대한 Prometheus 메트릭(allowed_traffic, denied_traffic 포함)을 조회하여 네트워크 트래픽 패턴을 파악합니다.
8. 네트워크 보안 구성 변경 타임스탬프와 보안 정책 위반 타임스탬프를 24시간 이내로 비교하고, 네트워크 정책 이벤트를 보조 증거로 사용하여 네트워크 변경이 보안 정책을 위반하는지 확인합니다.

## 진단

1. 1단계와 3단계의 네트워크 정책 구성을 검토합니다. 정책이 모든 소스에서의 인그레스를 허용하거나 이그레스가 제한되지 않으면 최우선 보안 문제입니다. 강화가 필요한 규칙을 파악합니다.

2. 6단계의 보안 감사 결과를 분석합니다. HIGH 또는 CRITICAL 결과가 네트워크 보안과 관련되면 해결을 우선시합니다. 결과에서 정책 위반이 나타나면 영향을 받는 특정 리소스와 네임스페이스를 파악합니다.

3. 5단계의 네트워크 정책 컨트롤러 로그에서 의심스러운 트래픽 패턴이나 무단 접근 시도가 나타나면 보안 모니터링이 잠재적 위협을 탐지하고 있는 것입니다. 플래그된 트래픽의 출발지와 목적지를 조사합니다.

4. 7단계의 네트워크 트래픽 메트릭을 검토합니다. allowed_traffic 패턴에서 예상치 못한 접근 경로가 나타나면 의도적인 것인지 확인합니다. denied_traffic이 높으면 과도하게 제한적인 정책으로 인해 정상 트래픽이 차단될 수 있습니다.

5. 8단계의 구성 변경 타임스탬프가 보안 위반과 상관관계가 있으면 최근 변경이 문제를 도입한 것입니다. 해당 변경을 되돌리거나 수정하는 데 집중합니다.

분석이 결론에 도달하지 못하는 경우: 2단계의 이벤트에서 최근 네트워크 정책 이벤트를 확인합니다. 보안 위반이 특정 네임스페이스에 집중되어 있는지(국소적 구성 오류 시사) 또는 분산되어 있는지(정책 템플릿 문제 시사) 판단합니다. 네트워크 보안 모니터링이 위반을 탐지하고 알림을 보내도록 올바르게 구성되어 있는지 확인합니다.
