# Route 53 Latency-Based Routing Not Working - Route 53 지연 기반 라우팅 미작동

## 의미

Route 53 지연 기반 라우팅이 작동하지 않는 현상(라우팅 실패 또는 Route53LatencyRoutingFailed 경보 트리거)은 지연 기반 라우팅 정책이 구성되지 않았거나, 상태 확인 실패로 라우팅이 방해되거나, 지연 레코드가 잘못되었거나, 라우팅 정책 평가가 실패하거나, 엔드포인트 상태가 라우팅을 방해하거나, Route 53 상태 확인 구성이 잘못되었을 때 발생합니다.
 Route 53 지연 기반 라우팅이 실패하고, 트래픽이 지연 기반으로 라우팅되지 않으며, DNS 라우팅이 최적이 아닙니다. 이는 DNS 및 라우팅 계층에 영향을 미치며 라우팅 최적화를 감소시킵니다.

## 영향

Route 53 지연 기반 라우팅이 실패합니다. 트래픽이 지연 기반으로 라우팅되지 않습니다. DNS 라우팅이 최적이 아닙니다. 지연 기반 라우팅 정책이 효과가 없습니다. 사용자 요청이 최저 지연 엔드포인트로 라우팅되지 않습니다. 라우팅 성능이 저하됩니다.

## 플레이북

1. Route 53 호스팅 영역 `<hosted-zone-id>`가 존재하고 리전 `<region>`의 Route 53 AWS 서비스 상태가 정상인지 확인합니다.
2. Route 53 호스팅 영역 `<hosted-zone-id>`를 조회하여 지연 기반 라우팅 레코드, 레코드 구성, 라우팅 정책 설정을 점검하고, 지연 기반 라우팅이 구성되어 있는지 검증합니다.
3. 지연 기반 라우팅 레코드의 Route 53 상태 확인을 조회하여 상태 확인 상태, 상태 확인 구성, 엔드포인트 상태를 점검하고, 상태 확인이 통과하는지 검증합니다.
4. Route 53 쿼리 로그가 포함된 CloudWatch Logs 로그 그룹에서 지연 기반 라우팅 패턴이나 라우팅 결정 로그를 필터링하여 라우팅 결정 세부 정보를 확인합니다.
5. Route 53 호스팅 영역 `<hosted-zone-id>`의 CloudWatch 지표(HealthCheckStatus)를 최근 1시간 동안 조회하여 상태 확인 패턴을 파악하고, 상태 확인 상태를 분석합니다.
6. 지연 기반 라우팅 정책이 있는 Route 53 리소스 레코드 세트를 나열하고 레코드 구성, 엔드포인트 구성, 라우팅 정책 평가를 확인하여 레코드 구성을 검증합니다.
7. Route 53 호스팅 영역 `<hosted-zone-id>`의 지연 레코드를 조회하여 지연 리전 매핑을 확인하고, 지연 리전이 올바르게 구성되어 있는지 점검합니다.
8. Route 53 상태 확인 구성을 조회하여 상태 확인 설정이 엔드포인트 요구사항과 일치하는지 확인하고, 상태 확인 구성이 라우팅에 영향을 미치는지 점검합니다.
9. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹에서 최근 24시간 이내 호스팅 영역 `<hosted-zone-id>` 관련 Route 53 레코드 또는 상태 확인 변경 이벤트를 필터링하여 구성 변경 사항을 확인합니다.

## 진단

1. Route 53 상태 확인의 CloudWatch 지표(플레이북 5단계)를 분석하여 최근 1시간 동안의 HealthCheckStatus를 포함한 엔드포인트 상태 패턴을 파악합니다. 엔드포인트에 대한 상태 확인이 실패하면 지연에 관계없이 해당 엔드포인트로 트래픽을 라우팅할 수 없습니다.

2. Route 53 상태 확인 상태 및 구성(플레이북 3단계)을 검토하여 지연 기반 레코드와 연관된 모든 엔드포인트의 상태 확인이 통과하는지 확인합니다.

3. Route 53 쿼리 로그가 포함된 CloudWatch Logs(플레이북 4단계)를 확인하여 라우팅 결정 패턴을 파악합니다. 로그에서 클라이언트 위치에 관계없이 트래픽이 지속적으로 단일 엔드포인트로 라우팅되면 다른 엔드포인트의 상태 확인이 실패하거나 지연 레코드가 잘못 구성되었을 수 있습니다.

4. 지연 기반 라우팅 레코드 구성(플레이북 2단계)을 확인하여 적절한 지연 리전과 세트 식별자로 레코드가 올바르게 정의되어 있는지 검증합니다.

5. 지연 레코드 리전 매핑(플레이북 7단계)을 검토하여 각 엔드포인트가 지연 측정을 위한 올바른 AWS 리전과 연결되어 있는지 확인합니다.

6. CloudTrail 이벤트(플레이북 9단계)와 라우팅 실패 타임스탬프를 5분 이내로 상관 분석하여 레코드 또는 상태 확인 변경을 파악합니다.

7. 상태 확인 구성(플레이북 8단계)을 분석하여 상태 확인 설정이 엔드포인트 요구사항과 일치하는지 확인합니다. 상태 확인 경로, 포트 또는 프로토콜이 잘못되면 엔드포인트가 잘못 비정상으로 표시될 수 있습니다.

상관관계를 찾을 수 없는 경우: 기간을 7일로 확장하고, 엔드포인트 지연 측정 및 라우팅 정책 평가 로직을 포함한 대안적 증거 소스를 검토합니다.
