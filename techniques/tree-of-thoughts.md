# Tree-of-Thoughts (ToT) — 분기 탐색 추론

## 개념

하나의 추론 경로가 아닌 여러 가지 사고 분기를 동시에 탐색하고,
각 분기를 평가하여 가장 유망한 경로를 선택하는 기법.
Yao et al. (2023) 논문에서 제안. CoT가 단일 경로라면 ToT는 트리 구조로 확장한 것이다.
BFS(너비 우선) 또는 DFS(깊이 우선) 전략으로 탐색할 수 있다.

## 핵심 원리

- 탐색 공간 확장: CoT의 단일 경로 한계를 극복하여 여러 가능성을 동시에 고려
- 자기 평가(Self-evaluation): 각 분기의 유망성을 LLM 스스로 평가하여 가지치기(pruning)
- 백트래킹: 막다른 경로에서 되돌아와 다른 경로를 탐색할 수 있음
- Game of 24, 창의적 글쓰기 등에서 CoT 대비 큰 성능 향상 (74% vs 4%)

## 프롬프트 템플릿

```
다음 문제에 대해 Tree-of-Thoughts 방식으로 분석하세요.

**문제:** [문제 설명]

**1단계 — 가능한 접근법 도출 (분기 생성):**
- 접근법 A: [설명]
- 접근법 B: [설명]
- 접근법 C: [설명]

**2단계 — 각 접근법 평가 (가지치기):**
| 접근법 | 실현 가능성 (1-5) | 효과 (1-5) | 리스크 (1-5) | 종합 |
|--------|------------------|-----------|-------------|------|
| A | | | | |
| B | | | | |
| C | | | | |

**3단계 — 최적 경로 선택 및 심화:**
선택: [접근법 X]
이유: [평가 근거]

**4단계 — 선택한 경로 내 하위 분기:**
- 세부 방안 X-1: [...]
- 세부 방안 X-2: [...]
→ 최종 선택: [X-N] 이유: [...]
```

## 실전 예시

**시나리오: 레거시 모놀리스 → 마이크로서비스 마이그레이션 전략**

접근법 A: Big Bang 전환 → 평가: 리스크 5, 효과 4 → 탈락
접근법 B: Strangler Fig 패턴 → 평가: 리스크 2, 효과 4 → 유망
접근법 C: 신규 기능만 MSA → 평가: 리스크 1, 효과 2 → 보수적

→ B 선택 후 하위 분기:
B-1: API Gateway 기반 라우팅 → 구현 복잡도 중
B-2: 이벤트 기반 분리 → 데이터 일관성 이슈
→ B-1 선택 (점진적 전환에 적합)

## 변형 및 조합

- **ToT + Self-Consistency**: 각 분기에서 다수결 적용
- **ToT + SWOT**: 각 분기를 SWOT으로 평가
- **ToT + Devil's Advocate**: 유망한 분기에 반론 제기
- **ToT + Decomposition**: 각 분기를 하위 문제로 분해

## 주의사항

- 단순한 문제에는 과도한 오버헤드 — CoT로 충분한 경우가 많음
- 분기가 너무 많으면 평가 자체가 부정확해질 수 있음 (3-5개가 적정)
- LLM의 자기 평가가 항상 정확하지 않음 — 외부 검증 필요
- 토큰 소비가 CoT의 3-5배 이상

## 적합한 상황

전략적 의사결정, 아키텍처 설계, 복잡한 트레이드오프 분석, 창의적 문제 해결, 마이그레이션 계획
