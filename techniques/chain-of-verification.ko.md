# Chain-of-Verification (CoVe) — 자기 검증 체인

## 개념

LLM이 자신의 답변을 생성한 뒤, 검증 질문을 스스로 만들어
각 주장을 하나씩 확인하고 수정하는 기법. Dhuliawala et al. (2023) 제안.
환각(hallucination) 감소에 특히 효과적이며, 사실 기반 답변의 신뢰도를 높인다.

## 핵심 원리

- 메타인지 시뮬레이션: "내가 한 말이 맞는지 스스로 확인"하는 과정
- 환각 감소: 리스트 기반 질문에서 환각률을 최대 50% 감소
- 독립적 검증: 검증 질문에 답할 때 원래 답변을 참조하지 않아야 편향 방지
- 4단계 프로세스: 초기 답변 → 검증 질문 생성 → 독립 검증 → 최종 수정

## 프롬프트 템플릿

```
**1단계 — 초기 답변 생성:**
[질문]에 대한 답변: [초기 답변]

**2단계 — 검증 질문 생성:**
위 답변에서 검증이 필요한 주장을 추출하고 각각에 대한 검증 질문을 만드세요:
1. 주장: "[주장 1]" → 검증 질문: "[이것이 사실인가?]"
2. 주장: "[주장 2]" → 검증 질문: "[근거가 있는가?]"
3. 주장: "[수치/날짜]" → 검증 질문: "[정확한 값인가?]"

**3단계 — 독립 검증 (원래 답변을 보지 않고):**
1. [검증 질문 1] → 답: [독립적으로 확인한 결과]
2. [검증 질문 2] → 답: [독립적으로 확인한 결과]
3. [검증 질문 3] → 답: [독립적으로 확인한 결과]

**4단계 — 검증 결과 종합:**
- ✅ 확인됨: [일치하는 주장들]
- ⚠️ 수정 필요: [불일치 발견 — 원래 주장 vs 검증 결과]
- ❓ 불확실: [검증 불가능한 주장 — "확인 필요" 표시]

**5단계 — 최종 수정 답변:**
[검증 결과를 반영하여 수정된 최종 답변]
```

## 실전 예시

**시나리오: "Redis Cluster의 최대 노드 수는?"**

초기 답변: "Redis Cluster는 최대 1000개 노드를 지원하며, 16384개 해시 슬롯을 사용합니다."

검증 질문:
1. "Redis Cluster 최대 노드 수가 1000개인가?" → 검증: 공식 문서에 "최대 1000개 노드" 명시 ✅
2. "해시 슬롯이 16384개인가?" → 검증: 맞음, CRC16 mod 16384 ✅

→ 수정 불필요, 원래 답변 유지

## 변형 및 조합

- **CoVe + CoT**: 각 검증 단계를 CoT로 상세 추론
- **CoVe + Self-Consistency**: 검증 질문에 대해 다중 경로로 답변
- **CoVe + Role**: "팩트체커 역할로 검증하세요"
- **CoVe + Few-Shot**: 검증 예시를 포함하여 패턴 학습

## 주의사항

- 검증 질문 자체가 잘못될 수 있음 (garbage in, garbage out)
- "독립 검증"이라 해도 같은 LLM이 하면 같은 편향을 가질 수 있음
- 모든 주장을 검증하면 토큰 소비가 2-3배 증가
- 주관적 판단(의견, 추천)에는 사실 검증이 적용되지 않음

## 적합한 상황

사실 확인이 중요한 답변, 기술 문서 작성, 보고서, 환각 방지, 수치/날짜 포함 답변
