# Lateral Thinking — Thinking Outside the Frame

## Concept

Proposed by Edward de Bono (1967). A creative thinking method that approaches problems from unexpected directions rather than through logical (vertical) thinking. A technique for finding "different answers" rather than "the right answer." If vertical thinking is "digging deeper," lateral thinking is "digging a hole somewhere else."

## Key Principles

- Frame shifting: Change the very frame through which you view the problem
- Deliberate illogic: Innovation emerges from ideas that "don't make logical sense"
- Provocation: "PO (Provocative Operation)" — deliberately set impossible assumptions
- Random input: Discover new connections through unrelated stimuli
- Complementary to vertical thinking: Find ideas laterally, then validate vertically

## Prompt Template

```
**Problem:** [Problem to solve]

**Conventional approach (vertical thinking):**
[Typical methods attempted and their limitations]

**Applying lateral thinking techniques:**

**1. Reversal:**
"What if we completely flip the problem?"
- Original: [A → B] → Reversed: [B → A]
- Discovery: [...]

**2. Provocation (PO):**
"PO: [An assumption that seems impossible]"
- What system would be needed if this assumption were true?
- Discovery: [...]

**3. Random Entry:**
Unrelated word: '[random word]'
- Associations from this word: [...]
- Connection to the problem: [...]

**4. Fractionation:**
What if we split the problem in a completely different way?
- Existing split: [...]
- New split: [...]
- Discovery: [...]

**5. Analogy:**
What's similar in a completely different field?
- [Solution from another field] → Application: [...]

**Newly discovered approach:**
[Most promising idea from the lateral thinking exercise]

**Vertical validation:**
Is this idea feasible? [Logical verification]
```

## Practical Example

**Scenario: "Log storage costs are too high"**

Vertical: Log compression, shorter retention periods, cheaper storage
Lateral:
- Reversal: "What if we don't store logs at all?" → Real-time streaming analysis only, then discard
- Provocation: "PO: What if logs could judge their own importance?" → ML-based log importance classification
- Random ('library'): Library indexing system → Store only log metadata, retrieve originals on demand
- Discovery: 95% of logs are never queried → Hot/cold tiering + sampling

## Variations and Combinations

- **Lateral + Vertical**: Generate ideas laterally → validate vertically
- **Lateral + SCAMPER**: Combine with the systematic creativity technique
- **Lateral + Analogy**: Deliberately seek analogies from distant domains
- **Lateral + Brainstorming**: Apply lateral thinking rules in team brainstorming

## Caveats

- "Illogical" ideas may be rejected in business environments
- Separate idea generation from evaluation (no judgment during generation)
- Not every problem needs this — if conventional methods work, they're the best choice
- Lateral thinking alone isn't actionable — vertical validation is always required
