# KubeClientCertificateExpiration

## 의미

API Server 인증에 사용되는 클라이언트 인증서가 7일 이내(경고) 또는 24시간 이내(긴급)에 만료됩니다(KubeClientCertificateExpiration 알림 트리거). 인증서가 갱신 없이 만료 날짜에 근접하고 있습니다. 인증서 메타데이터에서 경고 또는 긴급 임계값 이내의 만료 날짜가 표시되고, 로그에 인증서 만료 오류가 나타나며, 인증서가 만료에 근접함에 따라 TLS 핸드셰이크 실패가 발생할 수 있습니다. 이는 인증 및 권한 부여 플레인에 영향을 미치며, 클라이언트의 클러스터 접근을 차단할 인증서 수명 주기 관리 문제를 나타냅니다. 일반적으로 실패한 인증서 갱신 프로세스, 잘못 구성된 인증서 수명 또는 인증서 관리 시스템 실패가 원인이며, 만료되는 인증서를 사용하는 애플리케이션에서 인증 오류가 발생할 수 있습니다.

## 영향

KubeClientCertificateExpiration 알림이 발생합니다. 만료 후 클라이언트가 클러스터와 상호작용할 수 없게 됩니다. Kubernetes API와 통신하는 클러스터 내 서비스가 저하되거나 사용 불가가 될 수 있습니다. 인증 실패가 발생하고, 로그에 인증서 만료 오류가 나타나며, TLS 핸드셰이크 실패로 API 접근이 차단됩니다. 서비스 계정 토큰 문제가 발생할 수 있고, Controller가 조정에 실패할 수 있습니다. 영향받는 클라이언트의 클러스터 운영이 차단되며, 인증서 만료 후 API 접근이 거부됩니다.

## 플레이북

1. 모든 CertificateSigningRequest 리소스를 나열하여 보류, 거부 또는 실패한 인증서 요청을 식별합니다.

2. kube-system 네임스페이스에서 마지막 타임스탬프 순으로 이벤트를 나열하여 최근 Control Plane 이벤트를 조회하고, 인증서 만료 경고, TLS 핸드셰이크 실패 또는 인증 오류를 필터링합니다.

3. 클러스터의 서비스 계정, Controller 또는 API 클라이언트가 사용하는 클라이언트 인증서의 만료 정보를 조회하고 7일 이내(경고) 또는 24시간 이내(긴급)에 만료되는 인증서를 식별합니다.

4. Pod와 Controller가 사용하는 ServiceAccount 리소스를 조회하고 서비스 계정 토큰 만료를 확인하여 토큰 만료가 인증서 만료와 일치하는지 검증합니다.

5. 클라이언트 인증서의 발급 및 만료 타임스탬프를 검증하여 남은 유효 기간을 판단하고 만료에 근접한 인증서를 식별합니다.

6. CertificateSigningRequest 리소스를 조회하고 인증서를 업데이트했어야 하는 인증서 갱신 또는 갱신 프로세스를 확인하여 실패한 갱신 시도를 식별합니다.

7. 만료되는 인증서를 사용하는 `<namespace>` 네임스페이스의 Pod `<pod-name>`에서 이벤트와 로그를 조회하고 'certificate expired', 'TLS handshake failure', 'authentication failed'를 포함한 인증서 관련 오류 패턴을 필터링합니다.

8. 인증 기관(CA) 구성을 포함하는 Secret 리소스를 조회하고 인증 기관(CA) 구성과 인증서 서명 요청(CSR) 프로세스를 검증하여 구성 문제를 식별합니다.

## 진단

1. 플레이북의 인증서 관련 이벤트를 분석하여 어떤 인증서가 만료에 근접하고 있고 현재 상태가 무엇인지 식별합니다. 이벤트에서 인증서 만료 경고 또는 TLS 핸드셰이크 준비 실패가 나타나면, 이벤트 타임스탬프를 사용하여 긴급도를 판단합니다.

2. 이벤트에서 보류 또는 실패한 CertificateSigningRequest가 나타나면, 플레이북 1단계의 CSR 상태를 검토합니다. CSR 이벤트에서 거부, 실패 또는 보류 상태가 나타나면, 인증서 갱신 프로세스가 실패하여 조사가 필요합니다.

3. 이벤트에서 인증서 갱신 프로세스 실패가 나타나면, 인증서 관리 시스템 상태를 검증합니다. 만료 경고 이전 타임스탬프에서 갱신 관련 이벤트가 오류 또는 실패를 보이면, 자동 갱신이 실패한 것입니다.

4. 이벤트에서 서비스 계정 토큰 문제가 나타나면, 플레이북 4단계의 서비스 계정 토큰 만료를 검증합니다. 토큰 만료가 인증서 만료 타임스탬프와 일치하면, 둘 다 함께 해결해야 합니다.

5. 이벤트에서 여러 클라이언트에 걸친 인증서 만료가 나타나면, 만료 패턴을 분석합니다. 여러 인증서가 유사한 타임스탬프에서 만료되면, 인증서 관리 시스템 실패 또는 일괄 발급 문제일 가능성이 높습니다.

6. 이벤트에서 CA 갱신 또는 업데이트 활동이 나타나면, CA 구성 영향을 검증합니다. 인증서 만료 경고 전에 CA 이벤트가 발생했다면, CA 변경이 인증서 유효성 또는 신뢰 체인에 영향을 미쳤을 수 있습니다.

7. 이벤트에서 발급 날짜 기준으로 예상보다 일찍 만료되는 인증서가 나타나면, 인증서 수명 구성을 검증합니다. 인증서의 수명이 예상보다 짧으면, 잘못 구성된 인증서 매개변수를 수정해야 합니다.

**연관 관계를 찾을 수 없는 경우**: 인증서 유효 기간까지 시간 범위를 확장하고, 인증서 갱신 구성을 검토하며, 인증서 관리 시스템 실패를 확인하고, CA 구성을 검증하며, 과거 인증서 수명 주기 패턴을 검토합니다. 인증서 만료는 즉각적인 운영 변경보다 잘못 구성된 인증서 수명, 실패한 갱신 프로세스 또는 인증서 관리 시스템 문제로 인해 발생할 수 있습니다.
