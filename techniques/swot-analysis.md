# SWOT Analysis — 강점·약점·기회·위협 분석

## 개념

Strengths(강점), Weaknesses(약점), Opportunities(기회), Threats(위협)의
4가지 축으로 상황을 분석하는 전략 프레임워크.
Albert Humphrey(1960s, 스탠포드 연구소)에서 체계화.
내부 요인(S/W)과 외부 요인(O/T)을 구분하여 전략을 도출한다.

## 핵심 원리

- 구조적 분석: 4가지 축으로 상황을 체계적으로 정리
- 내부/외부 구분: 통제 가능한 것(내부)과 불가능한 것(외부)을 분리
- 전략 도출: TOWS 매트릭스로 4가지 교차 전략 생성
- 커뮤니케이션: 경영진/이해관계자에게 상황을 간결하게 전달

## 프롬프트 템플릿

```
**분석 대상:** [시스템/프로젝트/기술/팀 설명]

**SWOT 매트릭스:**

| | 긍정적 (Helpful) | 부정적 (Harmful) |
|---|--------|--------|
| **내부 (Internal)** | **S — Strengths (강점)** | **W — Weaknesses (약점)** |
| | 1. [...] | 1. [...] |
| | 2. [...] | 2. [...] |
| | 3. [...] | 3. [...] |
| **외부 (External)** | **O — Opportunities (기회)** | **T — Threats (위협)** |
| | 1. [...] | 1. [...] |
| | 2. [...] | 2. [...] |
| | 3. [...] | 3. [...] |

**TOWS 전략 도출:**

| 전략 | 설명 | 구체적 행동 |
|------|------|-----------|
| **SO (강점×기회)** | 강점을 활용하여 기회를 극대화 | [...] |
| **WO (약점×기회)** | 약점을 보완하여 기회를 활용 | [...] |
| **ST (강점×위협)** | 강점으로 위협에 대응 | [...] |
| **WT (약점×위협)** | 약점과 위협을 최소화 (방어적) | [...] |

**우선순위:**
1. [가장 중요한 전략]: [이유]
2. [두 번째 전략]: [이유]
```

## 실전 예시

**시나리오: "팀의 모니터링 시스템 SWOT"**

S: Prometheus+Grafana 전문성, 자체 알림 규칙 라이브러리
W: 분산 트레이싱 부재, 로그 중앙화 미흡, 대시보드 표준화 안 됨
O: OpenTelemetry 성숙, 관리형 서비스(Datadog, Grafana Cloud) 비용 하락
T: 마이크로서비스 전환으로 관찰 대상 10배 증가, 규정 준수 요구 강화

SO: Prometheus 전문성 + OTel 도입 → 통합 관측성 플랫폼 구축
WO: 관리형 서비스로 분산 트레이싱/로그 약점 보완
ST: 자체 알림 규칙으로 MSA 전환 시 모니터링 커버리지 유지
WT: 최소한의 규정 준수 대시보드 우선 구축

## 변형 및 조합

- **SWOT + MECE**: 각 축의 항목을 MECE로 검증
- **SWOT + Six Hats**: Yellow=S/O, Black=W/T, Green=전략 도출
- **SWOT + Hypothesis**: 각 전략을 가설로 세우고 검증 계획 수립
- **SWOT + OKR**: TOWS 전략을 OKR로 변환

## 주의사항

- 항목 나열에 그치고 전략 도출을 하지 않으면 무의미
- 내부/외부 구분이 모호한 경우가 있음 (예: 기술 부채는 내부? 외부?)
- 정적 분석이므로 시간에 따른 변화를 반영하기 어려움
- 주관적 판단이 많이 개입됨 — 여러 사람의 의견을 종합해야 함

## 적합한 상황

기술 선택, 프로젝트 평가, 경쟁 분석, 마이그레이션 결정, 팀 역량 평가, 전략 수립
