# Lateral Thinking — 수평적 사고

## 개념

Edward de Bono(1967) 제안. 논리적(수직적) 사고가 아닌,
기존 프레임을 벗어나 예상치 못한 방향에서 접근하는 창의적 사고법.
'정답'이 아닌 '다른 답'을 찾는 기법. 수직적 사고가 "더 깊이 파는 것"이라면
수평적 사고는 "다른 곳에 구멍을 파는 것"이다.

## 핵심 원리

- 프레임 전환: 문제를 바라보는 틀 자체를 바꿈
- 의도적 비논리: 논리적으로 "말이 안 되는" 아이디어에서 혁신이 나옴
- 도발(Provocation): "PO(Provocative Operation)" — 의도적으로 불가능한 가정을 세움
- 무작위 입력: 관련 없는 자극을 통해 새로운 연결 발견
- 수직적 사고와 상호보완: 수평적으로 아이디어를 찾고, 수직적으로 검증

## 프롬프트 템플릿

```
**문제:** [해결할 문제]

**기존 접근 (수직적 사고):**
[일반적으로 시도하는 방법과 그 한계]

**수평적 사고 기법 적용:**

**1. 역전 (Reversal):**
"문제를 완전히 뒤집으면?"
- 원래: [A → B] → 역전: [B → A]
- 발견: [...]

**2. 도발 (Provocation — PO):**
"PO: [불가능해 보이는 가정]"
- 이 가정이 사실이라면 어떤 시스템이 필요한가?
- 발견: [...]

**3. 무작위 입력 (Random Entry):**
무관한 단어: '[랜덤 단어]'
- 이 단어에서 연상되는 것: [...]
- 문제와 연결하면: [...]

**4. 분할 (Fractionation):**
문제를 완전히 다른 방식으로 쪼개면?
- 기존 분할: [...]
- 새로운 분할: [...]
- 발견: [...]

**5. 유추 (Analogy):**
전혀 다른 분야에서 비슷한 것은?
- [다른 분야의 해결책] → 적용: [...]

**발견된 새로운 접근:**
[수평적 사고에서 나온 아이디어 중 가장 유망한 것]

**수직적 검증:**
이 아이디어가 실현 가능한가? [논리적 검증]
```

## 실전 예시

**시나리오: "로그 저장 비용이 너무 높다"**

수직적: 로그 압축, 보존 기간 단축, 저렴한 스토리지 사용
수평적:
- 역전: "로그를 저장하지 않으면?" → 실시간 스트리밍 분석만 하고 버리기
- 도발: "PO: 로그가 스스로 중요도를 판단한다면?" → ML 기반 로그 중요도 분류
- 무작위('도서관'): 도서관의 색인 시스템 → 로그 메타데이터만 저장, 원본은 온디맨드 조회
- 발견: 로그의 95%는 한 번도 조회되지 않음 → 핫/콜드 티어링 + 샘플링

## 변형 및 조합

- **Lateral + Vertical**: 수평적으로 아이디어 생성 → 수직적으로 검증
- **Lateral + SCAMPER**: 체계적 창의 기법과 결합
- **Lateral + Analogy**: 먼 도메인에서 의도적으로 유추 탐색
- **Lateral + Brainstorming**: 팀 브레인스토밍에서 수평적 사고 규칙 적용

## 주의사항

- 비즈니스 환경에서 "비논리적" 아이디어가 거부될 수 있음
- 아이디어 생성과 평가를 분리해야 함 (생성 시 판단 금지)
- 모든 문제에 적용할 필요 없음 — 기존 방법이 통하면 그것이 최선
- 수평적 사고만으로는 실행 불가 — 반드시 수직적 검증 필요

## 적합한 상황

창의적 문제 해결, 브레인스토밍, 혁신, 막힌 문제 돌파, 비용 최적화, 프로세스 혁신
