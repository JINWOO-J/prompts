# Meta-Prompting — 프롬프트를 만드는 프롬프트

## 개념

LLM에게 프롬프트 자체를 생성하거나 개선하도록 요청하는 기법.
프롬프트 엔지니어링의 자동화. 더 나은 프롬프트를 LLM이 스스로 설계한다.
Suzgun & Kalai (2024) "Meta-Prompting" 논문에서 체계화.

## 핵심 원리

- 프롬프트 최적화 자동화: 수동 시행착오 대신 LLM이 최적 프롬프트를 탐색
- 전문가 지식 불필요: 프롬프트 엔지니어링 경험이 없어도 좋은 프롬프트 생성 가능
- 반복 개선: 생성된 프롬프트를 테스트하고 피드백으로 개선하는 루프
- 도메인 적응: 특정 도메인에 맞는 프롬프트를 자동 생성

## 프롬프트 템플릿

```
당신은 프롬프트 엔지니어링 전문가입니다.

**목표:** [달성하고 싶은 것]
**대상 LLM:** [사용할 모델 — GPT-4, Claude 등]
**입력 데이터:** [프롬프트에 들어갈 데이터 유형]
**출력 요구사항:** [원하는 출력 형식, 길이, 톤]
**제약 조건:** [반드시 포함/제외할 요소]

위 목표를 달성하기 위한 최적의 프롬프트를 작성하세요.

프롬프트에 포함할 요소:
1. 명확한 역할 설정 (Role)
2. 구체적인 지시사항 (Task)
3. 출력 형식 지정 (Format)
4. 예시 (Few-Shot, 필요 시)
5. 제약 조건 (Constraints)
6. 사용된 기법 설명

---
**생성된 프롬프트:**
[여기에 프롬프트 작성]

**설계 근거:**
- 사용된 기법: [CoT, Role, Few-Shot 등]
- 이 구조를 선택한 이유: [...]
- 예상 효과: [...]
- 개선 가능한 점: [...]
```

## 실전 예시

**시나리오: "코드 리뷰 프롬프트를 만들어줘"**

목표: PR의 코드를 리뷰하여 버그, 성능, 보안 이슈를 찾는 프롬프트
대상: Claude Sonnet
출력: 이슈 목록 (심각도, 위치, 설명, 수정 제안)

→ 생성된 프롬프트에 Role(시니어 리뷰어) + 체크리스트(OWASP Top 10) +
출력 형식(테이블) + Few-Shot(좋은 리뷰 예시) 포함

## 변형 및 조합

- **Meta + Self-Refinement**: 생성된 프롬프트를 자기 비평 후 개선
- **Meta + A/B Testing**: 여러 프롬프트를 생성하고 비교 테스트
- **Meta + Domain Expert**: 도메인 전문가 피드백을 반영하여 재생성

## 주의사항

- 생성된 프롬프트가 항상 최적은 아님 — 수동 검토 필요
- 메타 프롬프트 자체의 품질이 결과를 좌우
- 과도하게 복잡한 프롬프트를 생성할 수 있음 (단순화 지시 필요)
- 특정 모델에 최적화된 프롬프트가 다른 모델에서는 비효율적일 수 있음

## 적합한 상황

프롬프트 최적화, 반복 작업 자동화, 프롬프트 템플릿 생성, 도메인별 프롬프트 설계
