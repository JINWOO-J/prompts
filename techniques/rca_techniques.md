# RCA 특화 프롬프트 기법 정리

> `found.md` 기반으로 RCA(근본 원인 분석)에 직접 활용 가능한 기법을 선별·정리한 문서.

---

## 📌 RCA 프로세스 단계별 기법 선택 가이드

```
장애 발생
    │
    ▼
[1단계] 초기 범위 설정        →  Step-back 프롬프팅
    │
    ▼
[2단계] 원인 탐색              →  5-Whys + CoT (단순 장애)
    │                              ToT (복합/다중 원인 장애)
    ▼
[3단계] 가설 수립 및 검증      →  가설 검증 (Hypothesis Testing)
    │
    ▼
[4단계] 타임라인 재구성        →  프롬프트 체이닝 (4단계 파이프라인)
    │
    ▼
[5단계] (보안 침해) 공격 분석  →  Diamond Model + MITRE ATT&CK 연계
    │
    ▼
[6단계] RCA 보고서 검증        →  CoVe + Self-Refinement
```

---

## 🔍 기법 상세 설명

### 1. Step-back 프롬프팅 — 초기 범위 설정

| 항목 | 내용 |
|------|------|
| **목적** | 세부 로그 분석 전, 상위 원칙/패턴 먼저 파악 |
| **적합 상황** | 원인 범주가 불명확할 때 (네트워크? 앱? DB?) |
| **핵심 기법** | "이 장애를 분석하기 전에, [시스템 유형]에서 이 증상이 나타나는 일반적인 원인 패턴은 무엇인가?" |
| **프롬프트 파일** | `rca/02_hypothesis_stepback.md` |

---

### 2. 5-Whys + CoT (Chain-of-Thought) — 원인 탐색 (기본)

| 항목 | 내용 |
|------|------|
| **목적** | "인적 오류" 수준에서 멈추지 않고 프로세스/정책 실패까지 파고들기 |
| **적합 상황** | 인과 관계가 비교적 선형적인 단일 시스템 장애 |
| **핵심 기법** | 단계별 "왜?" × 5회, 각 단계 논리 명시 |
| **프롬프트 파일** | `rca/01_basic_rca.md` |

---

### 3. ToT (Tree-of-Thoughts) — 원인 탐색 (고급)

| 항목 | 내용 |
|------|------|
| **목적** | 2~4개의 독립 원인 가설 경로를 동시에 탐색, 위험도·영향도 점수화 |
| **적합 상황** | 멀티 시스템 장애, 원인 후보가 복수일 때 |
| **핵심 기법** | 각 경로별 확률, 영향 범위, 검증 방법을 병렬로 제시하게 유도 |
| **프롬프트 파일** | `rca/04_advanced_tot.md` |

---

### 4. 가설 검증 (Hypothesis Testing) — 가설 수립 및 검증

| 항목 | 내용 |
|------|------|
| **목적** | 2~3개의 가설 선제 수립 후, 각 가설 입증/반박에 필요한 증거 목록 생성 |
| **적합 상황** | 데이터가 부족한 초기 단계, 다양한 원인 후보 좁히기 |
| **핵심 기법** | "각 가설을 입증하려면 어떤 로그/지표를 확인해야 하는가?" |
| **프롬프트 파일** | `rca/02_hypothesis_stepback.md` |

---

### 5. 프롬프트 체이닝 — 타임라인 재구성

| 항목 | 내용 |
|------|------|
| **목적** | 복잡한 RCA를 단계별 파이프라인으로 분할 처리 |
| **적합 상황** | 멀티 서비스 장애, 로그 소스가 다양한 경우 |
| **파이프라인** | `로그 분류 → 이벤트 타임라인 → 근본원인 → 재발방지 액션` |
| **프롬프트 파일** | `rca/03_timeline_chain.md` |

---

### 6. Diamond Model + MITRE ATT&CK 연계 — 보안 침해 분석

| 항목 | 내용 |
|------|------|
| **목적** | 보안 인시던트의 공격 경로와 IOC를 표준화된 프레임워크로 분석 |
| **적합 상황** | 보안 침해, 이상 행위 탐지, APT 분석 |
| **출력 형식** | Diamond Model 4축 표 + ATT&CK Tactic/Technique ID 매핑 |
| **프롬프트 파일** | `rca/05_security_rca.md` |

---

### 7. CoVe + Self-Refinement — RCA 보고서 검증

| 항목 | 내용 |
|------|------|
| **목적** | 초안의 논리적 오류·사실 오류 자가 검증 후 최종본 생성 |
| **적합 상황** | 대외 공유 RCA 보고서, Post-mortem 문서 작성 시 |
| **핵심 기법** | 초안 작성 → 검증 질문 생성 → 비판 → 수정 루프 |
| **프롬프트 파일** | `rca/06_quality_review.md` |

---

### 8. Context Scaffolding — 컨텍스트 주입 (공통 적용)

| 항목 | 내용 |
|------|------|
| **목적** | 환경 정보를 구체적으로 주입하여 환각 억제 |
| **모든 프롬프트에 포함** | 시스템 구성, 사용 기술 스택, 장애 발생 시각, 영향 범위 |
| **가드레일** | "첨부된 로그/런북 데이터 내에서만 답변, 모르면 '데이터 부족' 응답" |
