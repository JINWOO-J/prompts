# First Principles Thinking — 제1원리 사고 (머스크 사고)

## 개념

기존 가정과 관행을 모두 제거하고, 가장 근본적인 진실(물리 법칙, 수학적 사실 등)부터
재구성하는 사고법. Elon Musk가 SpaceX 로켓 비용 절감에 활용한 것으로 유명.
아리스토텔레스의 "제1원리(First Principles)"에서 유래.
"배터리 팩이 kWh당 600달러인 이유는?" → 원재료 비용은 80달러 → 직접 조달하면 된다.

## 핵심 원리

- 유추에 의한 사고(Reasoning by Analogy)의 한계 극복: "남들이 하니까"가 아닌 근본부터 재구성
- 혁신의 원천: 대부분의 혁신은 기존 가정을 깨는 데서 시작
- 비용 구조 분해: 복잡한 비용을 원재료/노동/시간으로 분해하면 최적화 포인트가 보임
- 인지 편향 제거: 앵커링, 현상 유지 편향, 밴드왜건 효과 등을 의식적으로 제거

## 프롬프트 템플릿

```
다음 문제를 제1원리 사고법으로 분석하세요.

**문제/현상:** [현재 상황 설명]

**1단계 — 기존 가정 나열:**
"우리가 당연하게 받아들이고 있는 것들은?"
- 가정 1: [...]
- 가정 2: [...]
- 가정 3: [...]

**2단계 — 각 가정 검증:**
| 가정 | 분류 | 근거 |
|------|------|------|
| 가정 1 | 사실 / 관행 / 편견 | [...] |
| 가정 2 | 사실 / 관행 / 편견 | [...] |
| 가정 3 | 사실 / 관행 / 편견 | [...] |

**3단계 — 근본 진실만 추출:**
확인된 변경 불가능한 진실:
1. [물리적/논리적 제약]
2. [수학적 사실]

**4단계 — 근본 진실로부터 재구성:**
이 진실들만으로 문제를 처음부터 다시 설계하면:
[새로운 해결책]

**5단계 — 비교:**
| 항목 | 기존 방식 | 제1원리 방식 | 차이 |
|------|----------|------------|------|
| 비용 | | | |
| 복잡도 | | | |
| 리스크 | | | |
```

## 실전 예시

**시나리오: "CI/CD 파이프라인이 왜 30분이나 걸리는가?"**

기존 가정: "빌드는 원래 오래 걸린다", "전체 테스트를 돌려야 안전하다"

검증:
- "빌드는 원래 오래 걸린다" → 관행. 실제 컴파일은 2분, 나머지는 의존성 다운로드
- "전체 테스트를 돌려야 한다" → 편견. 변경된 모듈의 테스트만 돌려도 95% 커버

근본 진실: 변경된 코드만 빌드/테스트하면 됨, 의존성은 캐시 가능

재구성: 증분 빌드 + 의존성 캐시 + 영향 범위 기반 테스트 → 30분 → 4분

## 변형 및 조합

- **First Principles + 5 Whys**: 가정을 검증할 때 "왜?"를 반복
- **First Principles + Inversion**: "이 가정이 틀리면 어떻게 되는가?"
- **First Principles + Decomposition**: 문제를 원자 단위로 분해
- **First Principles + MECE**: 근본 진실을 MECE로 분류

## 주의사항

- 시간이 많이 소요됨 — 모든 문제에 적용하면 비효율적
- "근본 진실"이라고 생각한 것이 실은 가정일 수 있음 (메타 검증 필요)
- 기존 시스템과의 호환성을 무시하면 실행 불가능한 해결책이 나올 수 있음
- 팀/조직의 제약(정치, 문화)은 "근본 진실"에 포함해야 함

## 적합한 상황

비용 최적화, 아키텍처 재설계, 혁신적 접근이 필요한 경우, 기술 부채 해소, 프로세스 개선
