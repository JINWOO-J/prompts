# Inversion Thinking — 역발상 사고

## 개념

'어떻게 성공할까' 대신 '어떻게 하면 확실히 실패할까'를 먼저 생각하는 기법.
찰리 멍거(Charlie Munger)가 즐겨 사용: "Tell me where I'm going to die, so I'll never go there."
야코비(Carl Jacobi)의 수학적 역전 원리에서 유래: "Invert, always invert."
실패 원인을 미리 제거하여 성공 확률을 높인다.

## 핵심 원리

- 실패 회피 > 성공 추구: 큰 실패를 피하는 것이 작은 성공을 쌓는 것보다 효과적
- 인지 편향 극복: 성공에 대한 낙관 편향을 역전으로 상쇄
- 구체적 행동 도출: "성공하려면"은 모호하지만 "실패하려면"은 구체적
- 리스크 관리의 핵심: 다운사이드를 먼저 제거하면 업사이드는 자연스럽게 따라옴

## 실전 예시

**시나리오: "SRE 팀의 온콜 체계를 확실히 망하게 하려면?"**

1. 문서화를 하지 않는다 → 신규 온콜러가 매번 처음부터 헤맴
2. 에스컬레이션 경로를 정의하지 않는다 → 심각한 장애에서 우왕좌왕
3. 알림을 전부 켜놓는다 → 알림 피로로 진짜 장애를 놓침
4. 포스트모템을 하지 않는다 → 같은 장애가 반복
5. 온콜 로테이션 없이 한 사람에게 몰아준다 → 번아웃

역전환: 런북 작성, 에스컬레이션 매트릭스, 알림 튜닝, 포스트모템 문화, 공정한 로테이션

## 변형 및 조합

- **Inversion + Pre-mortem**: 실패 시나리오를 더 체계적으로 분석
- **Inversion + First Principles**: 실패 원인을 근본까지 파고들기
- **Inversion + Devil's Advocate**: 역발상 결과에 다시 반론 제기
- **Inversion + Checklist**: 실패 회피 전략을 체크리스트로 만들기

## 주의사항

- 지나치게 부정적 사고에 빠질 수 있음 — 균형 필요
- 모든 실패 요인을 나열하면 분석 마비(analysis paralysis)
- 실패 회피만으로는 혁신적 성공을 이끌기 어려움
- 팀에서 사용 시 부정적 분위기로 흐르지 않도록 퍼실리테이션 필요

## 적합한 상황

리스크 관리, 프로젝트 계획, 장애 예방, 전략 검증, 투자 결정, 체크리스트 작성
