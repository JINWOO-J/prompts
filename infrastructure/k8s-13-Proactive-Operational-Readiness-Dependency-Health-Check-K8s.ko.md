# Dependency Health Check — 의존성 상태 점검

## 의미

의존성 상태 점검은 서비스 의존성 상태를 확인할 수 없거나 의존성 상태 문제가 감지되었음을 나타냅니다(DependencyHealthCheckFailed 또는 DependencyUnhealthy 같은 알림 발생).
 의존성 상태 점검이 실패하거나, 의존성 상태가 사용 불가하거나, 의존성 상태 모니터링에서 문제가 감지되거나, 의존성 연결 점검이 실패하거나, 의존성 상태 점검 도구가 사용 불가한 것이 원인입니다. 의존성 상태 점검에서 실패가 나타나고, 의존성 상태가 사용 불가하며, 의존성 상태 모니터링에서 문제가 표시되고, 의존성 상태 점검 도구가 실패합니다. 이는 의존성 관리 계층과 서비스 신뢰성에 영향을 미치며, 일반적으로 의존성 서비스 비가용, 의존성 연결 문제, 의존성 상태 점검 도구 실패, 또는 의존성 상태 모니터링 문제로 인해 발생합니다. 의존성 상태가 컨테이너 워크로드에 영향을 미치면 컨테이너 서비스 의존성이 비정상이고 애플리케이션에서 의존성 관련 실패가 발생할 수 있습니다.

## 영향

DependencyHealthCheckFailed 알림 발생, DependencyUnhealthy 알림 발생, 의존성 상태 확인 불가, 의존성 상태 문제 감지, 서비스 신뢰성 저하, 의존성 관련 실패 가능. 의존성 상태 점검에서 실패가 나타나며, 의존성 상태가 컨테이너 워크로드에 영향을 미치면 컨테이너 서비스 의존성이 비정상이고, Pod 의존성이 사용 불가하며, 컨테이너 애플리케이션에서 의존성 관련 실패가 발생할 수 있습니다. 애플리케이션에서 서비스 의존성 실패 또는 상태 점검 실패가 발생할 수 있습니다.

## 플레이북

1. 네임스페이스 <namespace>의 Service <service-name>을 상세 조회하여 엔드포인트 상태와 의존성 서비스 가용성을 확인합니다.

2. 네임스페이스 <namespace>의 최근 이벤트를 타임스탬프 순으로 조회하여 최근 의존성 상태 문제나 서비스 실패를 확인합니다.

3. 네임스페이스 <namespace>의 Endpoints <service-name>을 YAML 출력으로 조회하여 백엔드 Pod 연결 및 상태를 확인합니다.

4. 네임스페이스 <namespace>에서 app=health-check 레이블의 서비스 상태 점검 Pod 로그를 조회하고 의존성 상태 점검 실패를 필터링합니다.

5. 지난 24시간 동안의 서비스 상태에 대한 Prometheus 메트릭(healthy_endpoint_count, unhealthy_endpoint_count 포함)을 조회하여 의존성 상태 문제를 확인합니다.

6. 네임스페이스 <namespace>의 Ingress를 상세 조회하여 의존성 서비스 엔드포인트의 상태 점검 결과를 확인합니다.

7. 서비스 `<service-name>`의 서비스 메시 트레이스 데이터를 조회하고 트레이스 오류율을 분석하여 비정상 다운스트림 의존성을 확인합니다.

8. 의존성 서비스에 대한 활성 Prometheus 알림(firing 상태)을 조회하고 알림 상태가 의존성 상태 문제를 나타내는지 확인합니다.

## 진단

1. 1단계와 3단계의 서비스 엔드포인트 상태를 검토합니다. 엔드포인트에서 비정상 백엔드나 누락된 주소가 나타나면 의존성 서비스가 사용 불가합니다. 백엔드 Pod 상태와 네트워크 정책을 확인합니다.

2. 4단계의 상태 점검 로그를 분석합니다. 로그에서 상태 점검 실패 패턴이 나타나면 실패가 타임아웃 관련(성능 문제 시사)인지 오류 관련(서비스 문제 시사)인지 확인합니다.

3. 5단계의 상태 메트릭에서 높은 비정상 엔드포인트 수가 나타나면 여러 의존성이 실패하고 있습니다. 의존성 중요도에 따라 조사 우선순위를 정합니다.

4. 7단계의 트레이스 수준 의존성 상태를 검토합니다. 트레이스 분석에서 특정 다운스트림 서비스의 높은 오류율이 나타나면 해당 서비스에 조사를 집중합니다.

5. 8단계의 알림에서 의존성 서비스에 대해 발생하고 있으면 알림이 문제를 정확히 감지하고 있습니다. 문제가 있는데 알림이 발생하지 않으면 알림 임계값 조정이 필요합니다.

분석이 결론에 도달하지 못하는 경우: 2단계의 이벤트에서 의존성 상태 문제나 서비스 실패를 확인합니다. 6단계의 Ingress 상태를 검토하여 외부 엔드포인트 상태를 확인합니다. 상태 문제가 모든 의존성에 영향을 미치는지(네트워크 또는 인프라 문제 시사) 또는 특정 의존성에만 영향을 미치는지(서비스별 문제 시사) 판단합니다.
