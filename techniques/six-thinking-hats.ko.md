# Six Thinking Hats — 6색 모자 사고법

## 개념

Edward de Bono(1985) 제안. 6가지 색상의 모자가 각각 다른 사고 방식을 대표.
한 번에 하나의 관점에만 집중하여 병렬적으로 사고하는 기법.
회의에서 참가자 전원이 같은 모자를 쓰고 같은 방향으로 사고하면
갈등 없이 다각도 분석이 가능하다. 전통적 토론(대립)의 대안.

## 핵심 원리

- 병렬 사고: 모든 참가자가 동시에 같은 방향으로 사고 → 대립 감소
- 완전한 커버리지: 6가지 관점을 모두 거치므로 누락 방지
- 감정의 합법화: Red Hat으로 직관과 감정을 공식적으로 다룸
- 시간 효율: 각 모자에 시간 제한을 두면 회의 시간 단축
- 6가지 모자: White(사실), Red(감정), Black(비판), Yellow(긍정), Green(창의), Blue(관리)

## 프롬프트 템플릿

```
**주제:** [분석할 주제]

**🤍 White Hat (사실과 데이터):**
"객관적 사실만 봅시다."
- 알려진 사실: [...]
- 필요하지만 없는 데이터: [...]
- 수치/통계: [...]
- 출처와 신뢰도: [...]

**🔴 Red Hat (감정과 직관):**
"느낌을 솔직하게 말합시다. 근거 불필요."
- 첫인상/직감: [...]
- 불안한 점: [...]
- 기대되는 점: [...]
- 팀의 전반적 분위기: [...]

**⚫ Black Hat (비판과 리스크):**
"무엇이 잘못될 수 있는지 봅시다."
- 위험 요소: [...]
- 실패 가능성: [...]
- 약점과 결함: [...]
- 과거 유사 사례의 실패 원인: [...]

**🟡 Yellow Hat (긍정과 가치):**
"어떤 가치와 이점이 있는지 봅시다."
- 장점과 이점: [...]
- 기회: [...]
- 최선의 시나리오: [...]
- 장기적 가치: [...]

**🟢 Green Hat (창의와 대안):**
"새로운 아이디어를 자유롭게 냅시다."
- 새로운 아이디어: [...]
- 대안적 접근: [...]
- 기존 방식의 변형: [...]
- "만약 ~라면?" 시나리오: [...]

**🔵 Blue Hat (정리와 프로세스):**
"지금까지의 논의를 정리합시다."
- 각 모자의 핵심 포인트 요약: [...]
- 합의된 사항: [...]
- 미해결 이슈: [...]
- 다음 단계와 액션 아이템: [...]
```

## 실전 예시

**시나리오: "쿠버네티스 도입 여부 결정"**

White: 현재 트래픽 월 100만 요청, 서버 5대, 배포 주 2회, 장애 월 1회
Red: 팀이 K8s에 대한 기대감이 높지만 운영 부담에 대한 불안도 있음
Black: 학습 곡선 6개월, 운영 복잡도 3배 증가, 소규모 팀에 과도할 수 있음
Yellow: 자동 스케일링, 셀프힐링, 배포 자동화로 장기적 운영 효율 향상
Green: K8s 대신 ECS Fargate? 또는 K8s를 관리형(EKS)으로?
Blue: 결론 — 관리형 K8s(EKS)로 시작, 3개월 파일럿 후 전면 도입 결정

## 변형 및 조합

- **Six Hats + SWOT**: Yellow/Black을 SWOT의 S/W/O/T에 매핑
- **Six Hats + Devil's Advocate**: Black Hat을 Devil's Advocate로 강화
- **Six Hats + Role**: 각 모자를 다른 전문가 역할로 수행
- **Six Hats + Time-boxing**: 각 모자에 5분 제한으로 효율적 회의

## 주의사항

- 모자 순서가 결과에 영향을 줄 수 있음 (Blue → White → Red → Black → Yellow → Green → Blue 권장)
- Red Hat(감정)을 불편해하는 문화에서는 효과 제한적
- 혼자 사용 시 각 모자의 관점 전환이 어려울 수 있음
- Black Hat에 과도하게 시간을 쓰면 부정적 분위기로 흐를 수 있음

## 적합한 상황

팀 의사결정, 프로젝트 평가, 전략 수립, 브레인스토밍, 회의 퍼실리테이션
