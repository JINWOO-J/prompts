# 프롬프트 05: 보안 침해 RCA (Diamond Model + MITRE ATT&CK)

## 사용 방법
보안 인시던트(침해 의심, 이상 행위, APT 탐지) 상황에서 활용.  
`[대괄호]` 안의 내용을 실제 인시던트 정보로 교체하세요.  
**외부 로그를 붙여넣을 때는 반드시 `<AnalysisData>` 태그 내에 넣어 인젝션을 방지하세요.**

## 활용 팁

- **Human-in-the-Loop 원칙**: 호스트 격리, 계정 잠금 등 고위험 조치는 AI 분석 후 **반드시 담당자 승인**을 거쳐 실행하세요.
- **데이터 무결성**: 외부에서 수집한 로그는 항상 `<AnalysisData>` 태그 안에 배치하여 프롬프트 인젝션을 방지하세요.
- ATT&CK ID 매핑 후 **MITRE ATT&CK Navigator**에 시각화하면 팀 공유에 효과적입니다.
