# S3 Lifecycle Policy Not Deleting Objects - S3 수명 주기 정책 객체 미삭제

## 의미

S3 수명 주기 정책이 예상대로 객체를 삭제하지 않는 현상(스토리지 비용 문제 또는 S3LifecyclePolicyNotWorking 경보 트리거)은 수명 주기 정책 규칙이 구성되지 않았거나, 전환 규칙이 잘못되었거나, 만료 규칙이 충족되지 않았거나, 정책 상태가 비활성화되었거나, 객체 태그가 수명 주기 규칙 필터와 일치하지 않거나, S3 버킷 버전 관리가 현재 버전의 삭제를 방해할 때 발생합니다.
 S3 객체가 자동으로 삭제되지 않고, 스토리지 비용이 증가하며, 수명 주기 전환이 발생하지 않습니다. 이는 스토리지 및 비용 관리 계층에 영향을 미치며 스토리지 비용을 증가시킵니다. 일반적으로 정책 구성 문제, 규칙 필터 문제 또는 버전 관리 충돌이 원인이며, Intelligent-Tiering과 함께 S3 수명 주기를 사용하는 경우 정책 동작이 다를 수 있고 애플리케이션에서 스토리지 비용 문제가 발생할 수 있습니다.

## 영향

S3 객체가 자동으로 삭제되지 않습니다. 스토리지 비용이 증가합니다. 수명 주기 전환이 발생하지 않습니다. 만료 규칙이 적용되지 않습니다. 스토리지 최적화가 실패합니다. 오래된 객체가 축적됩니다. 수명 주기 정책 자동화가 효과가 없습니다. 비용 관리 목표가 충족되지 않습니다. S3LifecyclePolicyNotWorking 경보가 발생할 수 있으며, Intelligent-Tiering과 함께 S3 수명 주기를 사용하는 경우 정책 동작이 다를 수 있습니다. 애플리케이션에서 증가된 스토리지 비용이 발생할 수 있으며, 스토리지 최적화 목표가 충족되지 않을 수 있습니다.

## 플레이북

1. S3 버킷 `<bucket-name>`이 존재하고 리전 `<region>`의 S3 AWS 서비스 상태가 정상인지 확인합니다.
2. 리전 `<region>`의 S3 버킷 `<bucket-name>`을 조회하여 수명 주기 정책 구성, 규칙 정의, 만료 설정, 전환 규칙을 점검하고, 수명 주기 정책이 활성화되어 있는지 검증합니다.
3. S3 서버 접근 로그가 포함된 CloudWatch Logs 로그 그룹에서 버킷 `<bucket-name>` 관련 수명 주기 전환 이벤트나 삭제 패턴을 필터링하여 수명 주기 작업 세부 정보를 확인합니다.
4. S3 버킷 `<bucket-name>`의 CloudWatch 지표(BucketSizeBytes, NumberOfObjects)를 최근 30일 동안 조회하여 객체 축적 패턴을 파악하고, 스토리지 추세를 분석합니다.
5. 버킷 `<bucket-name>`에서 수명 주기 만료 규칙보다 오래된 생성 날짜를 가진 S3 객체를 나열하고 삭제되어야 했는지 확인하여 객체 수명을 분석합니다.
6. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹에서 버킷 `<bucket-name>` 관련 S3 수명 주기 정책 변경 이벤트를 필터링하여 정책 변경 사항을 확인합니다.
7. S3 버킷 `<bucket-name>`의 버전 관리 구성을 조회하여 버전 관리 상태를 확인하고, 버전 관리가 현재 버전의 삭제를 방해하는지 점검합니다.
8. S3 버킷 `<bucket-name>`의 수명 주기 정책 규칙 필터를 조회하여 객체 태그 일치를 확인하고, 태그 필터가 삭제를 방해하는지 점검합니다.
9. CloudTrail 이벤트가 포함된 CloudWatch Logs 로그 그룹에서 최근 30일 이내 버킷 `<bucket-name>` 관련 S3 객체 태깅 또는 수명 주기 정책 규칙 변경 이벤트를 필터링하여 태그 또는 규칙 변경 사항을 확인합니다.

## 진단

1. S3 버킷의 CloudWatch 지표(플레이북 4단계)를 분석하여 최근 30일 동안의 BucketSizeBytes 및 NumberOfObjects를 포함한 객체 축적 패턴을 파악합니다. 객체 수와 크기가 감소해야 할 때 증가하고 있으면 수명 주기 정책이 예상대로 실행되지 않고 있는 것입니다.

2. S3 수명 주기 정책 구성(플레이북 2단계)을 검토하여 만료 규칙이 올바르게 정의되고 활성화되어 있는지 확인합니다. 규칙 상태가 "Disabled"이면 객체가 삭제되지 않습니다. 만료 일수가 예상보다 높게 설정되어 있으면 객체가 아직 만료 수명에 도달하지 않았을 수 있습니다.

3. S3 서버 접근 로그(플레이북 3단계)를 확인하여 수명 주기 전환 또는 삭제 이벤트를 파악합니다. 로그에서 만료되어야 할 객체에 대한 수명 주기 작업이 표시되지 않으면 규칙 필터와 만료 설정을 확인합니다. 로그에서 일부 삭제는 표시되지만 전부는 아닌 경우 규칙 필터가 특정 객체를 제외하고 있을 수 있습니다.

4. 버킷 버전 관리 구성(플레이북 7단계)을 확인하여 버전 관리가 삭제 동작에 영향을 미치는지 판단합니다. 버전 관리가 활성화된 경우 수명 주기 규칙은 현재 버전과 이전 버전 모두에 대한 작업을 명시적으로 정의해야 합니다. 현재 버전을 삭제하면 삭제 마커가 생성되지만 이전 버전은 제거되지 않습니다.

5. 접두사 및 태그 필터를 포함한 수명 주기 정책 규칙 필터(플레이북 8단계)를 확인합니다. 객체가 규칙 필터(잘못된 접두사 또는 필요한 태그 누락)와 일치하지 않으면 수명 주기 규칙에 의해 처리되지 않습니다.

6. 삭제되어야 할 객체(플레이북 5단계)를 분석하여 생성 날짜가 만료 임계값을 초과하는지 확인합니다. 객체가 만료 기간보다 새로우면 아직 삭제 대상이 아닙니다.

7. CloudTrail 이벤트(플레이북 6단계 및 9단계)와 객체 축적 타임스탬프를 상관 분석하여 만료 규칙을 비활성화하거나 변경했을 수 있는 수명 주기 정책 변경을 파악합니다.

8. 30일 이내 여러 객체 접두사에 걸친 삭제 패턴을 비교합니다. 일부 접두사에서는 삭제가 발생하지만 다른 접두사에서는 발생하지 않으면 수명 주기 규칙에 잘못된 접두사 필터가 있거나 일부 접두사가 규칙에서 제외되었을 수 있습니다.

9. 수명 주기 정책 실행은 즉시 이루어지지 않습니다. S3는 객체가 대상이 된 후 수명 주기 규칙을 실행하는 데 최대 24~48시간이 걸릴 수 있습니다. 객체가 최근에 만료 대상이 되었다면 수명 주기 실행을 위한 충분한 시간을 허용합니다.

상관관계를 찾을 수 없는 경우: 기간을 90일로 확장하고, 수명 주기 정책 규칙 상태 및 객체 버전 관리 구성을 포함한 대안적 증거 소스를 검토하고, 정책 규칙 평가 지연이나 객체 버전 축적과 같은 점진적 문제를 확인하고, 수명 주기 정책 서비스 가용성과 같은 외부 종속성을 검증하고, 객체 보존의 과거 패턴을 점검합니다.
